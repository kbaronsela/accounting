# מפת מסכים (PWA) — MVP

מסמך מוצר-UI משלים ל-`architecture.md`. נתיבי URL מדויקים ייקבעו ביישום.

## 1. לקוח (`role = client`)

| מסך | מטרה |
|-----|------|
| השלמת הזמנה | טוקן מהמייל — הגדרת סיסמה ו/או התחברות OAuth (מותר רק אם המייל תואם להזמנה) |
| התחברות | Auth.js — משתמש קיים |
| בית | כפתור העלאה, מסמכים אחרונים |
| העלאה | מצלמה / בחירת קובץ / גרירה (דסקטופ) |
| עיבוד / איכות | הודעת טעינה; אם נכשל איכות — הנחיה לצלם שוב או העלאה חדשה |
| עריכה לפני שליחה | סכום בש״ח (מטבע קבוע), תאריך, ספק, הערה; ולידציה לפי `architecture.md` §7 |
| אישור ושליחה | פעולת `submit`; אישור שהמסמך נשלח |
| היסטוריה | רשימת מסמכים + צפייה (תצוגת קובץ / פרטים) |

אופציונלי: **הגדרות** — שפה (`locale`).

---

## 2. רואה חשבון (`role = accountant`)

| מסך | מטרה |
|-----|------|
| השלמת הזמנה | כמו לקוח — הזמנה מאדמין |
| התחברות | Auth.js |
| דשבורד | סיכום (לפי החלטת מוצר) |
| לקוחות | רשימת תיקים; יצירת תיק + הזמנת חבר ראשון; הוספת משתמש לתיק |
| הזמנות | אופציונלי: רשימת הזמנות פעילות, שליחה מחדש / ביטול |
| מסמכים | פילטרים: לקוח, סטטוס, טווחי תאריכים, סכומים |
| פירוט מסמך | קובץ (הורדה חתומה) + שדות `final_*` + `extracted` + מי העלה |
| הגדרות | שפה (`locale`), שינוי סיסמה כשמופעל |

## 3. אדמין (`role = admin`)
